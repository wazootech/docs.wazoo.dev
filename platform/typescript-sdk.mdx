---
title: "TypeScript SDK"
description: "Use @wazoo/api to manage Wazoo platform resources."
---

`@wazoo/api` is the TypeScript SDK for the Wazoo Platform API. It manages
platform resources; it does not replace `@worlds/client` for graph data
operations.

Use `@wazoo/api` for organizations, world metadata, platform tokens, world auth
tokens, usage, limits, and billing state. Use `@worlds/client` for import,
export, search, SPARQL, and graph updates.

Groups are postponed for v1. Worlds are direct children of Organizations.

## Install

```bash
npm install @wazoo/api
```

## Create a client

```typescript
import { createClient } from "@wazoo/api";

const wazoo = createClient({
  organization: "acme",
  token: process.env.WAZOO_PLATFORM_TOKEN!,
});
```

The token must be a platform token with the `wzp_` prefix.

You can pass either an organization ID or a canonical organization resource name:

```typescript
const wazoo = createClient({
  organization: "organizations/acme",
  token: async () => getPlatformToken(),
});
```

## Manage worlds

```typescript
const worlds = await wazoo.worlds.list();

const world = await wazoo.worlds.create({
  worldId: "support-knowledge",
  world: { displayName: "Support Knowledge" },
});
```

This creates platform metadata for a world. Use the Worlds Data API or
`@worlds/client` for graph data inside that world.

## Create a world auth token

```typescript
const authToken = await wazoo.worlds.createToken("support-knowledge");
```

World auth tokens are data-plane credentials with the `wzw_` prefix. Store them
separately from platform tokens.

## Read usage

```typescript
const usage = await wazoo.usage.get();
const limits = await wazoo.usage.limits();
const worldUsage = await wazoo.worlds.usage("support-knowledge");
```

Usage and limits use transparent resource accounting. Data-plane limit
enforcement is a separate integration point: world operations should report
resource usage, and the Platform API should aggregate it against organization
and world limits. If a configured resource limit would be exceeded, the API returns
`RESOURCE_EXHAUSTED`.

Internal ingestion can record transparent resource-accounting fields:

```typescript
await wazoo.usage.record({
  metric: "world.import.bytes",
  quantity: 1024,
  unit: "byte",
  worldId: "support-knowledge",
  providerCostMicrocents: 12,
});
```

## Sync status

```typescript
const { world, syncReport } = await wazoo.worlds.sync("support-knowledge");
```

`syncReport` describes whether the per-World Turso database was already healthy
or repaired by reconciliation.

## Billing state

Billing is part of the production Platform API. Stripe is the source of truth
for billing and subscription state, while Wazoo mirrors the entitlement state
needed for product behavior and limit enforcement.

The SDK should expose billing and entitlement operations when the server
supports the end-to-end Stripe-backed flow.

## Runtime

The SDK is a fetch-based JavaScript package with ESM and CommonJS builds. The
project is developed with Node.js and npm, but the SDK is intended to run in any
modern JavaScript runtime that provides `fetch`.
