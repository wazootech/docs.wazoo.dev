Worlds is Wazoo's flagship context engine. It stores facts as knowledge graphs,
exposes search and SPARQL query paths, and provides APIs that let agents
retrieve and update verifiable context. Instead of asking an LLM to infer facts
from model weights, Worlds lets the model use tools to retrieve facts from a
graph.

## What Worlds provides

- **Graph storage** - RDF triples and named graphs for durable facts.
- **Hybrid search** - Keyword and vector retrieval over graph literals.
- **SPARQL queries** - Declarative graph traversal and reasoning.
- **Deterministic grounding** - Tool calls let models query stored facts before
  generating language.
- **API and CLI surfaces** - Programmatic and terminal workflows for creating,
  importing, exporting, searching, and querying worlds.
- **Edge-ready adapters** - LibSQL/Turso and Deno KV backends for durable
  deployments.

## Quickstart

Start with the hosted API when you want managed storage, or use the same
TypeScript SDK locally when you want embedded memory during development.

### Prerequisites

1. Sign up at the [Worlds Console](https://console.wazoo.dev).
2. Generate an API key.
3. Save it as `WORLDS_API_KEY`.

### Install

<CodeGroup>
```bash Deno
deno add jsr:@wazoo/worlds-sdk
```

```bash npm
npx jsr add @wazoo/worlds-sdk
```

```bash Bun
bunx jsr add @wazoo/worlds-sdk
```

</CodeGroup>

### Run a graph loop

```typescript index.ts
import { Worlds } from "@wazoo/worlds-sdk";

const worlds = new Worlds({
  apiKey: process.env.WORLDS_API_KEY!,
  baseUrl: "https://api.wazoo.dev",
});

await worlds.create({
  slug: "quickstart-cloud",
  label: "My First World",
});

await worlds.import(
  "quickstart-cloud",
  `@prefix user: <https://example.org/user/> .
   @prefix wazoo: <https://wazoo.dev/#> .

   user:person wazoo:worksFor wazoo:organization .`,
  { contentType: "text/turtle" },
);

const result = await worlds.sparql(
  "quickstart-cloud",
  `PREFIX wazoo: <https://wazoo.dev/#>
   SELECT ?organization WHERE {
     <https://example.org/user/person> wazoo:worksFor ?organization
   }`,
);

console.log(JSON.stringify(result, null, 2));
```

For local or test workflows, use `LocalWorlds` from the same SDK. Switch to the
hosted `Worlds` client when you need managed persistence.
