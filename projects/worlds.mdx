Worlds is Wazoo's flagship context engine. It stores facts as knowledge graphs,
exposes search and SPARQL query paths, and provides APIs that let agents
retrieve and update verifiable context. Worlds lets models use tool calls to
retrieve facts from a graph rather than infer them from model weights.

Worlds is designed for curated memory, not transcript accumulation. Use
[Memory appraisal](/worlds/memory-appraisal) to decide what belongs in a world,
what belongs in another source system, and what should be discarded.

## What Worlds provides

- **Graph storage** - RDF triples and named graphs for durable facts.
- **Hybrid search** - Keyword and vector retrieval over graph literals.
- **SPARQL queries** - Declarative graph traversal and reasoning.
- **API and CLI surfaces** - Programmatic and terminal workflows for creating,
  importing, exporting, searching, and querying worlds.
- **Edge-ready adapters** - LibSQL/Turso and Deno KV backends for durable
  deployments.

## Quickstart

Start with the hosted API when you want managed storage, or use `LocalWorlds`
from the TypeScript SDK when you want embedded memory during development.

Hosted Worlds has two surfaces:

- `@wazoo/api` manages platform resources such as user-owned Worlds and tokens.
- `@worlds/client` works with graph data through create, import, export, search,
  and SPARQL operations.

### Prerequisites

1. Get access to the hosted Wazoo private beta.
2. Obtain a world token for the world you want to query.
3. Save it as `WORLDS_TOKEN`.

### Install

<CodeGroup>
```bash Deno
deno add jsr:@worlds/client
```

```bash npm
npx jsr add @worlds/client
```

```bash Bun
bunx jsr add @worlds/client
```

</CodeGroup>

### Run a graph loop

```typescript index.ts
import { Worlds } from "@worlds/client";

const worlds = new Worlds({
  apiKey: process.env.WORLDS_TOKEN!,
  baseUrl: "https://api.wazoo.dev",
});

await worlds.create({
  worldId: "quickstart-cloud",
  displayName: "My First World",
});

await worlds.import(
  "quickstart-cloud",
  `@prefix user: <https://example.org/user/> .
   @prefix wazoo: <https://wazoo.dev/#> .

   user:person wazoo:uses wazoo:worlds .`,
  { contentType: "text/turtle" },
);

const result = await worlds.sparql(
  "quickstart-cloud",
  `PREFIX wazoo: <https://wazoo.dev/#>
   SELECT ?resource WHERE {
     <https://example.org/user/person> wazoo:uses ?resource
   }`,
);

console.log(JSON.stringify(result, null, 2));
```

For local or test workflows, use `LocalWorlds` from the same SDK. Switch to the
hosted `Worlds` client when you need managed persistence.

To provision hosted resources programmatically, use the
[Platform TypeScript SDK](/platform/typescript-sdk).
