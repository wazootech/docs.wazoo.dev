title: "Hybrid search"
description: "Fusing vector embeddings with keyword full-text search for optimal retrieval."
---

Wazoo's search index combines **vector semantic similarity** and **keyword full-text search** (BM25/FTS5) via Reciprocal Rank Fusion (RRF).

## Retrieval modes

- **Hybrid (fused)** — Default when an embedding service is configured. RRF blends vector and keyword scores.
- **Keyword-only** — Pure FTS5 full-text search. Activates automatically when no embedding service is available.
- **Semantic-only** — Vector similarity search without keyword scoring.

## How it works

1. Content is chunked into searchable units.
2. Each chunk is indexed with:
   - `fts_value` — Subject label, predicate phrase, and literal text (for FTS and embedding).
   - `value` — The object literal returned as search result text.
   - `vector` — Embedding of the chunk (when an embedding service is configured).
3. Queries are matched against both indexes and fused via RRF (k=60).

## Graceful degradation

When the embedding service is unreachable, the search index automatically falls back to keyword-only mode, ensuring uninterrupted availability.

## Label predicates

Built-in label predicates (`rdfs:label`, `skos:prefLabel`, `schema:name`) are always indexed. Extend with custom label predicates via adapter options.
