---

The quad store is the storage layer for RDF quads (triples with an optional graph context).

## Interface

```typescript
interface QuadStoreInterface {
  getQuads(subject, predicate, object, graph): AsyncIterable<Quad>;
  putQuads(quads: Quad[]): Promise<void>;
  deleteQuads(quads: Quad[]): Promise<void>;
  hasQuads(subject, predicate, object, graph): Promise<boolean>;
}
```

## In-memory: `RdfjsQuadStore`

Uses `N3.Store` under the hood. No external dependencies — ideal for development and testing.

```typescript
import { RdfjsQuadStore } from "@worlds/client/quad-store";
```

## Adapter naming convention

| Suffix         | Implements        | File name pattern |
| -------------- | ----------------- | ----------------- |
| `*RdfjsStore`  | `rdfjs.Store`     | `rdfjs-store/*.ts` |
| `*QuadStore`   | `QuadStoreInterface` | `quad-store/*.ts` |

- `LibsqlQuadStore` → `LibsqlRdfjsStore`
- `DenokvQuadStore` → `DenokvRdfjsStore`
