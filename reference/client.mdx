---

The `Client` class orchestrates the quad store, search index, and SPARQL engine into a unified API.

## Constructor

```typescript
import { Client } from "@worlds/client";

const client = new Client(options?: ClientOptions);
```

### Options

| Option        | Type                     | Default     | Description |
| ------------- | ------------------------ | ----------- | ----------- |
| `quadStore`   | `QuadStoreInterface`     | `RdfjsQuadStore` | Graph storage backend |
| `searchIndex` | `SearchIndexInterface`   | `RdfjsSearchIndex` | Hybrid search index |
| `sparqlEngine`| `SparqlEngineInterface`  | Auto-detected | SPARQL query engine |

## Methods

### `client.import(data: string): Promise<void>`

Import RDF triples. Accepts Turtle, TriG, N-Triples, and other RDF serializations.

### `client.query(sparql: string): Promise<SparqlResponse>`

Execute a SPARQL query and return bindings.

### `client.search(query: string, options?): Promise<SearchResult[]>`

Hybrid search over indexed content. Returns results with `subject`, `predicate`, and `text`.

### `client.reindex(options?: ReindexOptions): Promise<void>`

Rebuild the search index. Useful after bulk imports or schema upgrades.

```typescript
await client.reindex({
  include: { graphs: ["http://example.org/ontology"] },
});
```
